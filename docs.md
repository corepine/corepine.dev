# Corepine Docs Maintainer Guide

This file explains how the docs engine works in this project, how to add or upgrade versions, and how links stay version-safe.

## 1. Architecture At A Glance

Request flow:

1. Route receives URL in [routes/web.php](/Users/namu/Desktop/workspace/corepine.dev/routes/web.php):
   - `/{project}/docs/{segments?}`
2. [DocsController](/Users/namu/Desktop/workspace/corepine.dev/app/Http/Controllers/DocsController.php) resolves project/version/slug through [DocsService](/Users/namu/Desktop/workspace/corepine.dev/app/Services/DocsService.php).
3. Markdown is rendered by [DocsRenderer](/Users/namu/Desktop/workspace/corepine.dev/app/Services/DocsRenderer.php).
4. Renderer rewrites docs links and builds heading TOC.
5. Output is shown in [docs/show.blade.php](/Users/namu/Desktop/workspace/corepine.dev/resources/views/docs/show.blade.php), which extends [layouts/docs.blade.php](/Users/namu/Desktop/workspace/corepine.dev/resources/views/layouts/docs.blade.php).

## 2. Folder Structure

Docs content is package-first, then version-folder:

```text
resources/views/docs/{project}/{version_folder}/
```

Current example:

```text
resources/views/docs/modal/0_1x/
resources/views/docs/threads/0_1x/
```

Each version folder should contain:

- `_nav.json` (left sidebar sections/items)
- Markdown pages (`*.md`)

## 3. Single Source Of Truth Config

Project docs config lives in [config/docs.php](/Users/namu/Desktop/workspace/corepine.dev/config/docs.php).

For each project, configure:

- `label`
- `description`
- `default_slug`
- `navigation_file`
- `content_path`
- `versions` map

`versions` map format:

```php
'versions' => [
    '0_1x' => '0.1x',
    '0_2x' => '0.2x',
],
```

- Key = folder name
- Value = URL version label
- Order matters: newest should be last (used as latest)

## 4. Route + Version Behavior

The docs route is:

```text
/{project}/docs/{segments?}
```

Version handling:

- If the first segment is a known version label (example: `0.1x`), that exact version is used.
- If no version segment is present, latest version is used automatically.
- Latest version URLs are clean (no version prefix).
- Older version URLs include the version prefix.

Examples for project `modal` when latest is `0.1x`:

- Latest welcome: `/modal/docs`
- Latest page: `/modal/docs/quick-start`
- Older page (example `0.0x`): `/modal/docs/0.0x/quick-start`

This behavior is generated by `docs()->url(...)` and [DocsService::url()](/Users/namu/Desktop/workspace/corepine.dev/app/Services/DocsService.php).

## 5. How Markdown Linking Works

Link rewriting happens in [DocsRenderer::rewriteHref()](/Users/namu/Desktop/workspace/corepine.dev/app/Services/DocsRenderer.php).

Use these patterns inside markdown:

1. `doc:` links (recommended)

```md
[Installation](doc:installation)
[Theme tokens](doc:styling#theme-tokens)
```

2. Relative markdown links

```md
[Sibling page](./modal-attributes.md)
[Parent folder page](../installation.md)
```

3. Fragment-only links stay local:

```md
[Jump to API](#api)
```

What gets rewritten:

- `doc:*` links -> always to same project + current version context.
- Relative `.md` links -> resolved relative to current page slug, then converted to docs route.
- `.md` and `.markdown` extensions are stripped from route slugs.

What does not get rewritten:

- External links (`https://...`, `mailto:...`, etc.)
- Root-relative links (`/something`)
- Non-markdown file links (`.png`, `.pdf`, etc.)

Version-safe behavior example:

- Current URL: `/modal/docs/0.0x/quick-start`
- Markdown link: `[Setup](doc:setup)`
- Final URL: `/modal/docs/0.0x/setup`

Same link on latest docs page becomes:

- `/modal/docs/setup`

## 6. Sidebar + In-Page TOC

Left sidebar:

- Driven by `_nav.json` in each version folder.
- Order in `_nav.json` controls sidebar order.
- Every `slug` should match a markdown filename (without `.md`).

Right “On This Page” TOC:

- Generated from markdown headings.
- Depth controlled in [config/docs.php](/Users/namu/Desktop/workspace/corepine.dev/config/docs.php):
  - `toc.min_level`
  - `toc.max_level`

Current setup is H2 and H3 only:

```php
'toc' => [
    'min_level' => 2,
    'max_level' => 3,
],
```

If you want to include H4 later, change `max_level` to `4` in this one place.

## 7. Add A New Page

1. Create markdown file in the active version folder:
   - Example: `resources/views/docs/modal/0_1x/new-page.md`
2. Add entry to `_nav.json` with matching slug:
   - `"slug": "new-page"`
3. Link to it with `doc:new-page` from other pages.
4. Open route and verify:
   - `/modal/docs/new-page` (latest)

## 8. Upgrade To A New Docs Version

Use this checklist whenever you release a new docs version:

1. Copy the current version folder:
   - `resources/views/docs/modal/0_1x -> resources/views/docs/modal/0_2x`
2. Update [config/docs.php](/Users/namu/Desktop/workspace/corepine.dev/config/docs.php):
   - Add `'0_2x' => '0.2x'` to the `versions` map
   - Keep newest version last
3. Edit markdown files in the new folder only.
4. Keep old folder untouched for historical docs.
5. Verify URLs:
   - Latest docs should now resolve without prefix to `0.2x`
   - Old links with `/0.1x/...` should still work
6. Spot-check a few `doc:` and relative links from both versions.

## 9. Add A New Package Project

1. Create content folder:
   - `resources/views/docs/{new-project}/{version_folder}/`
2. Add `_nav.json` and initial markdown files.
3. Register project in [config/docs.php](/Users/namu/Desktop/workspace/corepine.dev/config/docs.php) under `projects`.
4. Visit:
   - `/{new-project}/docs`

No route file changes are needed for each new project because the route is dynamic.

## 10. Theme + Layout Notes

- Docs page shell is in [layouts/docs.blade.php](/Users/namu/Desktop/workspace/corepine.dev/resources/views/layouts/docs.blade.php).
- Landing page is [welcome.blade.php](/Users/namu/Desktop/workspace/corepine.dev/resources/views/welcome.blade.php).
- Theme toggle uses one button that cycles `light -> dark -> system`.
- Stored preference key: `localStorage.theme`.
- Legacy key `corepine-theme` is automatically migrated.

## 11. Quick Maintenance Checks

After docs changes, quickly verify:

1. `/` loads
2. `/modal/docs` loads
3. `/threads/docs` loads
4. `/modal/docs/quick-start` loads
5. `/threads/docs/installation` loads
6. One or two sidebar links load
7. Theme toggle cycles all 3 modes
8. `doc:` links keep version context on non-latest URLs
